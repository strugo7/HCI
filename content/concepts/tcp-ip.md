# TCP/IP

:::definition
TCP/IP הוא מחסנית הפרוטוקולים שהאינטרנט מיושם עליה בפועל, המחולקת לארבע שכבות —
Application, Transport, Internet ו-Link. כל שכבה פותרת בעיית תקשורת אחת ומספקת
שירות לשכבה שמעליה, כך שהעברת נתונים מקצה לקצה מתאפשרת מבלי שאף רכיב יצטרך לפתור
את כל הבעיות בעצמו.
:::

## הסבר פשוט

כששולחים הודעה ברשת צריך לפתור המון בעיות בו-זמנית: איך מייצגים אותה על הכבל, איך
מזהים את המחשב היעד, איך מוודאים שהכול הגיע, ואיך שני היישומים מבינים זה את זה.
TCP/IP מפרק את הבעיה הגדולה לארבע שכבות עצמאיות, כך שכל שכבה עוסקת בשלב אחד בלבד.

בזכות ההפרדה אפשר להחליף Wi-Fi בכבל בלי לגעת בדפדפן, ואפשר להמציא פרוטוקול חדש
בשכבת האפליקציה בלי לשנות אף נתב. זו גם הסיבה שכל מנגנון הגנה רואה רק את מה שהשכבה
שבה הוא פועל חושפת לו.

## הסבר טכני

ארבע השכבות, מלמעלה למטה, כפי שהן מוגדרות רשמית ב-IETF RFC 1122:

- **Application** — התוכן המשמעותי למשתמש ולאפליקציה: HTTP, HTTPS, DNS, SMTP, FTP.
  שכבה זו מקבילה לשכבות 5–7 של [[OSI Model]], ובאבטחה מכנים אותה "שכבה 7".
- **Transport** — מסירה בין תהליכים לפי מספרי פורט. TCP מספק חיבור אמין עם מספור
  ושידור מחדש; UDP מספק מסירה מהירה ללא חיבור וללא הבטחות. מקביל לשכבה 4 של OSI.
- **Internet** — כתובות IP וניתוב בין רשתות. IP מוסר מנות מהמקור ליעד ללא הבטחת
  מסירה. מקביל לשכבה 3 של OSI.
- **Link** — העברה בפועל על המדיום הפיזי: Ethernet, Wi-Fi. מקביל לשכבות 1–2 של OSI.

המנגנון שמחבר את השכבות הוא **Encapsulation**: כל שכבה עוטפת את המטען שקיבלה מלמעלה
בכותרת (header) משלה. בקשת HTTP הופכת ל-Segment (כותרת TCP), ל-Packet (כותרת IP)
ולבסוף ל-Frame (כותרת Ethernet) שנשלח על הכבל. בצד המקבל התהליך מתהפך שכבה-אחר-שכבה.

עטיפה זו היא הסיבה המבנית לכך ש-[[Firewall]] בשכבה 3 קורא את כותרות ה-IP וה-TCP —
המעטפה — אך לא את התוכן שבליבה. בחינת התוכן היא תפקידו של [[WAF]] בשכבה 7.

:::example
כשאתם טוענים אתר: הדפדפן יוצר בקשת HTTP בשכבת ה-Application; TCP בשכבת ה-Transport
עוטף אותה עם פורט 443; IP בשכבת ה-Internet מוסיף את כתובת השרת; ו-Ethernet בשכבת
ה-Link שולח את המסגרת אל הנתב. אותה הודעה עברה דרך כל ארבע השכבות בשבריר שנייה.
:::

:::warning
TCP/IP הוא המודל שהאינטרנט **מיישם**, ואילו OSI הוא מודל **ייחוס** בן שבע שכבות
שאיש אינו "מריץ". שני המודלים אינם מתחרים — המספור המקצועי (שכבה 3, שכבה 7) לקוח
מ-OSI, אך המנות עצמן בנויות לפי ארבע שכבות TCP/IP.
:::
