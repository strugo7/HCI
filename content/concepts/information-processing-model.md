# Information Processing Model (מודל עיבוד המידע)

:::definition
מודל עיבוד המידע מתאר את האדם כמעבד מידע: המוח מקבל **קלט** מהחושים, **מעבד** אותו (סינון בקשב, פירוש בתפיסה ואחסון בזיכרון), ומפיק **פלט** בצורת החלטה או פעולה.
:::

## הסבר פשוט

כל אינטראקציה עם ממשק עוברת שלושה שלבים: העין או האוזן קולטות גירוי (קלט), המוח מבין ומחליט מה לעשות איתו (עיבוד), והמשתמש מבצע פעולה (פלט). לכל שלב יש מגבלות, וכל שלב עלול להפוך ל"צוואר בקבוק" שעוצר את האינטראקציה.

## הסבר טכני

המונח **הכרה (Cognition)** הוא השם הכולל לתהליכי רכישת הידע, עיבודו, ייצוגו ויישומו במוח. המשמעות למעצב: המשתמש אינו "מצלמה" שקולטת הכול — הוא מערכת מוגבלת שמסננת, מפרשת ושוכחת. עיצוב טוב עובד עם מגבלות אלה ולא נגדן, ומחליק את המעברים בין השלבים במקום להעמיס על אחד מהם.

:::example
התחברות לאתר בנק: העין קולטת את שדות ההתחברות (קלט), המוח שולף את הסיסמה מהזיכרון ומפרש את מבנה הטופס (עיבוד), והאצבעות מקלידות ולוחצות "כניסה" (פלט). כשל בכל שלב — כפתור נסתר, סיסמה שנשכחה או כפתור לא ברור — עוצר את כל התהליך.
:::
