# Web Shell

:::definition
Web Shell הוא סקריפט זדוני המועלה לשרת web ומעניק לתוקף גישה ושליטה מרחוק על
השרת דרך ממשק web סטנדרטי (HTTP/HTTPS). אין לו מונח עברי מקובל — משתמשים במונח
האנגלי "Web Shell".
:::

## הסבר פשוט

תחשבו על שרת האתר של הארגון כעל בניין משרדים עם דלת כניסה אחת שכולם עוברים דרכה.
Web Shell הוא כמו מפתח סתר שהתוקף מצליח להשאיר בתוך הבניין — קובץ קטן שיושב בין
קובצי האתר הלגיטימיים ונראה כמו אחד מהם. מרגע שהוא שם, התוקף כבר לא צריך "לפרוץ"
שוב: הוא פשוט גולש לכתובת של הקובץ בדפדפן, מקליד פקודות, והשרת מבצע אותן בשבילו.

זהו לא סוג של נזק מיידי אלא **נקודת אחיזה קבועה** — דלת אחורית שדרכה התוקף חוזר
מתי שירצה, מריץ פקודות, קורא קבצים וגונב מידע, הכול דרך תעבורת web שנראית רגילה
לחלוטין.

## הסבר טכני

ל-Web Shell שלושה מאפיינים מגדירים:

- **Script-Based** — הוא נכתב בשפות צד-שרת שהשרת ממילא יודע להריץ: PHP, ASP,
  JSP או Python. אין צורך בקובץ הפעלה מהודר; מספיק קובץ טקסט שהשרת מפרש.
- **Hidden in Plain Sight** — הוא מוסווה בין הקבצים הלגיטימיים של האתר. השרת אמור
  להכיל קובצי סקריפט, ולכן קובץ נוסף אינו בולט.
- **Remote Control** — הוא מספק לתוקף הרצת פקודות, ניהול קבצים והוצאת מידע, כולם
  דרך בקשות HTTP/HTTPS רגילות.

המנגנון הזה הופך את ה-Web Shell לקשה במיוחד לזיהוי: הוא יושב במקום שבו קבצים
אמורים להיכתב, ומדבר עם התוקף על גבי אותו ערוץ (port 443) שבו עובר כל שאר תעבורת
ה-web. חומת אש היקפית לא תחסום אותו, מפני שהיא לא חוסמת HTTPS נכנס לשרת שכל
תפקידו לשרת HTTPS. ההגנה נגדו אינה בפרימטר אלא בניטור מתמשך של השרת עצמו — ראו
[[WAF]] ו-[[Defense in Depth]].

:::example
China Chopper הוא Web Shell מפורסם שגודלו כ-4KB בלבד — קטן דיו כדי להיטמע בין
מאות קבצים מבלי להיראות. קבוצות APT השתמשו בו נרחבות: התוקף מעלה אותו דרך טופס
העלאה פגיע, ומאותו רגע שולט בשרת דרך בקשות HTTP רגילות, בלי להפעיל שום חולשה
נוספת.
:::

:::warning
Web Shell המוסווה כקובץ לוג לגיטימי הוא מהקשים ביותר לזיהוי — לא בגלל תחכום הקוד,
אלא בגלל **המיקום והערוץ**: הקובץ יושב בספרייה שבה קבצים נכתבים ממילא, והוא נגיש
דרך HTTPS על port 443 שאף חומת אש אינה חוסמת. מנקודת מבט הרשת, שום דבר אינו נראה
חריג.
:::

:::diagram
תרשים שרת web: משתמש לגיטימי (חץ כחול) פונה דרך האינטרנט אל Normal Scripts על
השרת. תוקף (חץ אדום) פונה דרך האינטרנט אל ה-WebShell שיושב בין אותם קבצים בדיוק
("Hidden in Plain Sight"). מה-WebShell יוצאים שלושה חצים לתוצאות: אל DB Server
(יירוט וגניבת נתונים), אל Web Server (השחתת האתר והפניית משתמשים), ואל מחשב פנימי
(הזרקת קוד זדוני ותנועה רוחבית לרשת הפנימית).
:::
