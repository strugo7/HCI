# Masquerade

:::definition
התחזות (Masquerade) היא תת-התקפה של [[Deception]] שבה **ישות לא מורשית מתחזה לישות
מורשית** — היא מציגה את עצמה בפני המערכת כמי שאינה, ומקבלת את ההרשאות של הישות
האמיתית.
:::

## הסבר פשוט

התחזות עונה על השאלה: **מי שולח את ההודעה?**

המערכת מקבלת בקשה שנראית תקינה לחלוטין — הפורמט נכון, האסימון תקף, הבקשה סבירה.
הדבר היחיד שאינו נכון בה הוא **מי** שלח אותה. המערכת לא נפרצה במובן הקלאסי; היא
פשוט האמינה לזהות שהוצגה לה.

זו הסיבה שהתחזות כל כך קשה לגילוי: אין חבילה חריגה, אין קובץ זדוני, ואין תעבורה
חשודה. יש משתמש "לגיטימי" שעושה בדיוק את מה שמותר לו — רק שהוא לא הוא.

## הסבר טכני

המרצה מונה שישה וקטורים להתחזות (שקופית 31):

- **Phishing Attack** — [[Phishing]] מכוון לגניבת פרטי הזדהות.
- **Session Hijacking** — התוקף משתלט על session פעיל של משתמש ופועל בשמו.
- **Stolen Credentials** — שם משתמש וסיסמה שנגנבו ומשמשים לכניסה "רגילה".
- **API Key Theft** — ניצול מפתחות API גנובים לגישה לשירותים מוגבלים.
- **Fake Requests** — בקשות שנבנו כך שייראו כאילו הגיעו מגורם מורשה.
- **Social Engineering** — מניפולציה על משתמשים כדי שימסרו מידע רגיש בעצמם.

המשותף לכולם: **אף אחד מהם אינו משנה את הנתונים**. התוקף לוקח את הזהות, לא את
התוכן. לכן ההגנה על התחזות אינה עוברת דרך אימות תוכן אלא דרך חיזוק **אימות
הזהות** — המיטיגציה שהמרצה מייעד להתחזות היא **MFA** (אימות רב-שלבי), לצד
**Secure Credentials** ו-**Review Access** (שקופיות 42–43).

:::example
תרחיש **Impersonation of LLM Service via Stolen API Key** (שקופית 34): מפתח API של
שירות LLM דולף מתוך repository ציבורי. התוקף שולח באמצעותו בקשות לשירות, והמודל
מפיק תשובות **בשם המערכת**. מבחינת השירות, כל בקשה חתומה במפתח תקף של לקוח קיים.
:::

:::warning
התחזות אינה זיוף. ב-Masquerade **הישות** מזויפת והנתונים אמיתיים; ב-[[Falsification]]
**הנתונים** מזויפים והישות עשויה להיות אמיתית לחלוטין. זו ההבחנה שהמבחן חוזר
אליה.
:::

:::diagram
שתי דיאגרמות זו לצד זו, שתיהן מעל פס אופקי המסמן "Internet or other comms
facility".

משמאל — הזוג Bob / Lily / John (שקופית 32): Bob עומד ליד תווית "BOB PRETENDS TO
BE LILY", ומהחץ שיוצא ממנו אל John מוצמדת תווית שם "Lily". ההודעה עצמה תקינה.

מימין — הזוג Darth / Bob / Alice (שקופית 33): Darth שולח הודעה אל Alice, ומעל החץ
כתוב "Message from Darth that appears to be from Bob". Bob עצמו מצויר בצד, לא
פעיל — הוא כלל לא שלח דבר.

מתחת לשתי הדיאגרמות שורה אחת: "הנתונים אמיתיים — השולח מזויף".
:::
