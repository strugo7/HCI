# Active Attack

:::definition
תקיפה אקטיבית (Active Attack) היא תקיפה שבה התוקף מנסה **לשנות משאבי מערכת או
לשבש את פעולתה**. בלשון המרצה: "the attacker attempts to modify system resources
or disrupt operations, potentially leading to data breaches or system
malfunctions requiring immediate security measures".
:::

## הסבר פשוט

בתקיפה אקטיבית התוקף **נוגע** במערכת. הוא לא רק מסתכל — הוא כותב, מוחק, מזייף,
מציף או מנתק. משהו במערכת נראה אחרי התקיפה שונה ממה שנראה לפניה: רשומה שהשתנתה,
הודעה שלא נשלחה מעולם והתקבלה, שירות שאינו מגיב.

זו בדיוק הסיבה שהמרצה מוסיף להגדרה את המילים "requiring immediate security
measures". תקיפה אקטיבית מייצרת אפקט שאפשר להרגיש, ולכן היא מחייבת תגובה מיידית.

## הסבר טכני

המבחן שקובע אם תקיפה היא אקטיבית הוא מבחן אחד: **האם משאבי המערכת הושפעו?**
אם התשובה חיובית — התקיפה אקטיבית, בלי קשר לשאלה כמה מידע התוקף אסף תוך כדי.

שלוש מארבע הסטיות מזרימת המידע התקינה (**Normal Flow**) הן אקטיביות לפי המוסכמה
המקובלת:

- **Interruption** — הזרימה נעצרת ואינה מגיעה ליעד. פגיעה ב-[[Availability]].
- **Modification** — הזרימה מנותבת דרך התוקף, משתנה, וממשיכה ליעד. פגיעה
  ב-[[Integrity]].
- **Fabrication** — התוקף מזריק אל היעד זרימה חדשה שלא הייתה קיימת. פגיעה
  באותנטיקציה ובנאותות המידע.

היא **אינה** מוגדרת לפי כמות הנזק ולא לפי מורכבות הכלי. תוקף ששולח בקשה מזויפת
אחת ביצע תקיפה אקטיבית; תוקף שהאזין חודשיים למאות אלפי הודעות ולא נגע בדבר —
לא.

:::example
תוקף שהשתלט על ה-session בין שני רכיבי רשת תקינים, ומשתמש בו כדי לשלוח מסרים
מפוברקים שנראים לגיטימיים. הרכיב המקבל פועל לפי מסר שמעולם לא נשלח על ידי הרכיב
האמיתי — משאבי המערכת הושפעו, ולכן זו תקיפה אקטיבית.
:::

:::warning
"אקטיבי" אינו שם נרדף ל"חמור", ו"אקטיבי" בהחלט אינו שם נרדף ל"יירוט". המסיח
הקבוע במבחני העבר מצמיד "אקטיבי" ליירוט ו"פסיבי" לשיבוש — בדיוק ההפך מהמוסכמה.
יירוט טהור, שבו התוקף רק מאזין, הוא [[Passive Attack]]; שיבוש שירות משפיע על
משאבי המערכת ולכן הוא אקטיבי.
:::

:::warning
בשקופית 8 של המצגת המרצה מתייג את **Interception** כ-Active threat ואת
**Interruption** כ-Passive threat — הפוך מהמוסכמה ומההגדרה שהוא עצמו נתן בשקופית
7. ראו את הפירוט המלא בקובץ [[Passive Attack]] ובשיעור. אין להתעלם מהסתירה: יש
להכיר את שתי הגרסאות.
:::

וחץ ישר ביניהם. שלוש וריאציות אקטיביות מוצגות מתחתיה: (b) Interruption — החץ

שהמקור שלח דבר. בכל שלוש הווריאציות מסומן במפורש שמשאבי המערכת או המידע השתנו.
:::
