# סיכום — מודל הרשת: TCP/IP ושכבות התקשורת

:::summary
תקשורת רשת בנויה בשכבות כדי שכל שכבה תפתור בעיה אחת ותסתיר את פרטיה מזו שמעליה. שני
מודלים מתארים את החלוקה: OSI הוא מודל ייחוס בן שבע שכבות — השפה שבה מדברים באבטחה —
ואילו TCP/IP הוא המודל בן ארבע השכבות (Application, Transport, Internet, Link)
שהאינטרנט מיישם בפועל. המספור המקצועי, "שכבה 3" ו"שכבה 7", לקוח מ-OSI.

שלוש שכבות נושאות את כל הקורס: שכבה 3 (Network) מחזיקה את כתובות ה-IP ואת הניתוב;
שכבה 4 (Transport) מחזיקה את הפורטים ואת TCP/UDP; שכבה 7 (Application) מחזיקה את
התוכן — בקשת ה-HTTP וכל מה שבתוכה, כולל SQL Injection. המנגנון שמחבר ביניהן הוא
Encapsulation: כל שכבה עוטפת את המטען שקיבלה בכותרת משלה, כך שבקשת HTTP הופכת
ל-Segment, ל-Packet ולבסוף ל-Frame.

העטיפה הזו היא ההסבר המבני לכל הקורס: רכיב בשכבה 3 קורא רק את הכותרות החיצוניות —
המעטפה — ולעולם לא את התוכן שבליבה, ולכן הוא עיוור ל-SQL Injection. מכאן נגזר סדר
מנגנוני ההגנה: Packet Filter בודק בזול בשכבות 3–4, WAF/DPI בודק תוכן בשכבה 7 במחיר
ביצועים ופענוח TLS, ו-NGFW מאחד את שתיהן. השכבות אינן מתחרות — הן משלימות, וזהו
Defense in Depth במודל השכבות.
:::
