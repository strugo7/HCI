:::summary
Web Shell הוא סקריפט זדוני המושתל על שרת web ומעניק לתוקף שליטה מרחוק דרך ממשק
web רגיל (HTTP/HTTPS). בניגוד לווירוס או לתולעת שפועלים על תחנת הקצה, הוא יושב על
השרת עצמו ומהווה דלת אחורית קבועה: התוקף חוזר אליו מתי שירצה, בלי לפרוץ מחדש.
שלושת מאפייניו — מבוסס סקריפט, מוסתר בין הקבצים הלגיטימיים, ושליטה מרחוק — הם גם
מקור הסכנה וגם הסיבה שקשה לזהותו.

הוא מגיע לשרת דרך העלאת קבצים ללא אימות, ניצול חולשות כמו RFI/LFI ו-SQL injection,
פרטי גישה גנובים או שרשרת אספקה. לאחר ההשתלה התוקף מריץ פקודות, קוצר פרטי גישה,
מוציא מידע ובונה התמדה — אך היכולת המסוכנת ביותר היא תנועה רוחבית מה-Web Shell
ב-DMZ אל הרשת הפנימית. כאן מתחדדות כללי התעבורה של ה-DMZ: הגבלת מה שיוצא מהאזור
היא מה שמכיל פריצה של שרת בודד.

הנקודה שנבחנה ב-2024: Web Shell המוסווה כקובץ לוג לגיטימי הוא הקשה ביותר לזיהוי —
לא בגלל הקוד, אלא מפני שהוא יושב במקום שבו קבצים נכתבים ממילא, נגיש דרך port 443
שאף חומת אש אינה חוסמת, ותעבורתו זהה לתעבורת web רגילה. לכן לא תופסים אותו בפרימטר.
ההגנה היא רב-שכבתית (Defense in Depth): אימות העלאות, פיתוח מאובטח, הקשחת שרת,
סגמנטציה, WAF וניטור שלמות קבצים על השרת עצמו.
:::
