# סיכום — מתקפות מניעת שירות: DoS ו-DDoS

:::summary
מתקפת מניעת שירות (DoS) שוללת ממשתמשים לגיטימיים גישה לשירות בכך שהיא מציפה אותו
מעבר לקיבולתו — בלי לפרוץ פנימה, בלי לגנוב ובלי לשנות דבר. ב-DDoS המתקפה מבוזרת:
היא מגיעה בו-זמנית מעשרות אלפי מקורות, וזה ההבדל המהותי — הביזור מנטרל את ההגנה
הפשוטה, שכן חסימת כתובת IP אחת חסרת ערך מול רבים. מקור המקורות הוא [[Botnet]]:
רשת מכשירים נגועים (recruit → command → flood) הנשלטת דרך ערוץ
[[Command and Control]] — אותו מנגנון שלמדנו ביחידת הנוזקות.

המתקפות נחלקות לשלוש משפחות: volumetric (הצפת רוחב פס, נמדדת ב-Gbps/Tbps),
protocol (הכליית מצב חיבור, כמו SYN flood בנפח נמוך), ו-application-layer
(בקשות L7 שנראות כמו גולשים אמיתיים). טכניקת ה-amplification/reflection מנצלת
spoofing של כתובת המקור כדי לגייס שרתי DNS/NTP תמימים כמגברים המחזירים תעבורה אל
הקורבן.

הנקודה המרכזית: [[DDoS]] אינה דורשת חולשה. היא מנצלת את הקיבולת הסופית של
המערכת, לא פגם — ולכן שרת מוקשח ומעודכן ייפול בדיוק כמו מוזנח. זה מייחד אותה מכל
תקיפה אחרת בקורס. המיטיגציה אינה טלאי אלא ספיגה מבוזרת: rate limiting, scrubbing
centers ו-CDN/Anycast. חומת אש בשכבה 3 לבדה נכשלת — הקו נסתם לפניה, היא מסננת לפי
כתובת, והיא לא רואה תוכן L7. במשולש ה-[[CIA]] DDoS פוגעת ב-[[Availability]] בלבד,
ובטקסונומיית RFC 2828 היא הדוגמה הקנונית של [[Obstruction]].
:::
