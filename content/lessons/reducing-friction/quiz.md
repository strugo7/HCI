## Question

id: reducing-friction/q1
type: multiple-choice
difficulty: easy
cognitive: understand
estimatedTime: 45
points: 5
concepts:
  - microcopy
  - plain-language

What technique should a UX writer use to decide which reassuring microcopy to add to a screen?

### Answers

A. Add as much explanatory text as possible everywhere on the screen.

B. After presenting each piece of information, ask "what question will come up for the user next?" and answer it in the same place.

C. Avoid all reassuring text to keep the screen visually clean.

D. Move all reassurance to a separate terms-of-service document.

Correct: B
Explanation: הטכניקה המרכזית שנלמדה היא לשאול מראש מה השאלה הבאה של המשתמש ולענות עליה מיד באותו מקום, לפני שהוא נאלץ לשאול אותה. תיאור A מוביל לעומס טקסט מיותר וללא מיקוד. תיאור C מתעלם מהצורך האמיתי בהרגעה במקומות הנכונים. תיאור D מרחיק את המידע מהרגע שבו הוא באמת נחוץ למשתמש.
Learning Objective: להסביר מדוע שאילת "מה השאלה הבאה?" אחרי כל פיסת מידע מפחיתה חרדה ונטישה.
Misconception: סטודנטים לעיתים חושבים שיותר טקסט הרגעה תמיד טוב יותר, בלי להבין שהעיקרון הוא מיקוד ההרגעה בדיוק בשאלה הרלוונטית.

## Question

id: reducing-friction/q2
type: scenario
difficulty: medium
cognitive: analyze
estimatedTime: 60
points: 5
concepts:
  - microcopy
  - plain-language

What is the main problem with this microcopy?

### Scenario

A note-taking app shows the following text every time a user taps the standard "Save" button on a routine note: "Warning: this action may permanently and irreversibly alter your data in ways that cannot be undone."

### Answers

A. The text is too short to be taken seriously.

B. The text introduces a new, exaggerated fear into a routine, safe action that does not warrant it.

C. The text should be translated into more languages.

D. The button should be moved to a different part of the screen.

Correct: B
Explanation: שמירת פתק רגילה היא פעולה שגרתית ובטוחה לחלוטין, ולכן ניסוח דרמטי כל כך ("לצמיתות", "לא ניתן לביטול") מזריק חשש חדש שלא היה קיים ופוגע בביטחון המשתמש שלא לצורך. תיאורים A, C ו-D אינם מזהים את הבעיה המרכזית — חוסר ההתאמה בין חומרת הניסוח לחומרת הפעולה בפועל.
Learning Objective: לנתח מסך נתון ולזהות מיקרו-קופי שמזריק חשש חדש שלא היה קיים, במקום להרגיע חשש קיים.
Misconception: סטודנטים לעיתים חושבים שאזהרה חזקה תמיד "בטוחה יותר" מבחינה משפטית או עיצובית, בלי לשים לב לעלות שלה: עייפות אזהרות ואובדן אמון.

## Question

id: reducing-friction/q3
type: scenario
difficulty: medium
cognitive: apply
estimatedTime: 60
points: 5
concepts:
  - accessibility
  - plain-language

According to the principle of situational accessibility, what is most likely happening here?

### Scenario

A user is trying to read a long, technical error message in a banking app while driving with strong sunlight glaring on the screen. The user has no permanent visual impairment.

### Answers

A. The user will understand the message exactly as well as in any other situation, since they have no permanent disability.

B. The user's ability to read and understand text is temporarily reduced by the circumstances (glare, divided attention), so a complex message is likely to be missed.

C. Situational accessibility only applies to users with a permanent visual impairment.

D. The length of the message has no effect on comprehension in this situation.

Correct: B
Explanation: נגישות מצבית מתארת בדיוק תרחיש כזה — נסיבות זמניות (סנוור, קשב מפוצל בנהיגה) מפחיתות את יכולת הקריאה וההבנה גם ללא מוגבלות קבועה כלשהי. תיאור A מתעלם מהעיקרון המרכזי של נגישות מצבית. תיאור C מצמצם את המושג רק למוגבלות קבועה — בדיוק ההפך מהגדרתו. תיאור D שגוי, כי אורך ומורכבות ההודעה משפיעים ישירות על סיכויי ההבנה במצב של יכולת מופחתת.
Learning Objective: לזהות מצבים שבהם יכולת הקריאה של משתמש יורדת זמנית, גם ללא מוגבלות קבועה.
Misconception: סטודנטים רבים חושבים על נגישות רק כדרישה טכנית עבור בעלי מוגבלות רשומה, ומפספסים שהיא רלוונטית לכל משתמש במצבים מסוימים.

## Question

id: reducing-friction/q4
type: multiple-choice
difficulty: medium
cognitive: apply
estimatedTime: 60
points: 5
concepts:
  - plain-language

Which error message best follows the principles of plain language?

### Answers

A. "An error occurred during the validation process of the submitted request."

B. "The system detected a mismatch between the provided input and the required authentication format."

C. "Wrong password. Try again."

D. "Your request could not be processed due to non-compliance with system parameters."

Correct: C
Explanation: "Wrong password. Try again." קצר, ישיר, וללא ז'רגון טכני — מובן מיד בקריאה ראשונה, בדיוק עקרון השפה הנקייה. שאר האפשרויות משתמשות במונחים טכניים ("validation", "parameters", "authentication format") ובמשפטים ארוכים שמחייבים מאמץ פענוח מיותר, בניגוד לעיקרון מילים קצרות ומשפטים קצרים.
Learning Objective: ליישם את עקרונות השפה הנקייה (מילים קצרות, משפטים קצרים) כדי לשפר קטע מיקרו-קופי נתון.
Misconception: סטודנטים לעיתים חושבים שניסוח "מקצועי" ו"רשמי" תמיד עדיף בממשק, בלי להבין שהוא מגדיל את המאמץ הקוגניטיבי הנדרש מהמשתמש.
