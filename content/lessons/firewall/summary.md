# סיכום — חומת אש: סינון מנות וטבלאות כללים

:::summary
חומת אש (Firewall) היא שומר הסף בין הרשת הפנימית לרשת לא-מהימנה. בגרסה שהמרצה
מלמד — "Firewalls – layer 3", כלומר Packet Filtering — היא בוחנת כל מנה מול ארבעה
שדות בכותרת בלבד: Source IP, Destination IP, Port ו-Protocol, ולעולם לא את תוכן
המנה. לכן היא עוצרת גישה לפי מקור ופורט, אך לא נוזקה או SQL Injection המסתתרים
בתוך תעבורה מותרת — לשם כך נועד ה-WAF בשכבה 7.

הרכיב המרכזי הוא ה-ACL: טבלת כללים מסודרת שכל שורה בה מתירה (ALLOW) או חוסמת (DENY)
סוג מנה. חומת האש עוצרת בשורה הראשונה שמתאימה, ולכן הסדר מכריע. שתי מדיניויות ברירת
מחדל הפוכות: Implicit Deny (allow list / whitelist) חוסם כל מה שלא הותר במפורש — וזו
הגישה הבטוחה; Implicit Allow (block list / blacklist) מתיר כל מה שלא נחסם במפורש.

חומת אש מגיעה בכמה סוגי פריסה — Host-based (מחשב בודד), Network-based (רשת שלמה),
Stand-alone ו-Cloud — ובדרך כלל משלבים אותן. אך אף חומת אש היקפית אינה מספיקה
לבדה: היא שכבה אחת בלבד במערך Defense in Depth.
:::
