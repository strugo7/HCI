According to Don Norman's conceptual models framework, which component represents the physical product that the user actually interacts with and receives feedback from?

### Answers

A. The System Image — which includes the visual interface, documentation, and system feedback.

B. The Design Model — which outlines the database configurations and source code architectural rules.

C. The User's Mental Model — which represents the psychological state and goals of the end user.

D. The Usability Standard — which dictates the five dimensions of interface evaluation criteria.

Correct: A
Explanation: תמונת המערכת (System Image) היא הייצוג הפיזי והחזותי של המוצר שבו המשתמש נוגע, קורא ומקבל ממנו פידבק. המעצב אינו יכול להעביר את מודל העיצוב (Design Model) ישירות למשתמש, אלא רק לתווך אותו באמצעות תמונת המערכת כדי לעזור למשתמש לבנות מודל מנטלי נכון. התשובות האחרות מציגות חלקים אחרים של המודל הקונספטואלי.
Learning Objective: Define and distinguish the three conceptual models of Don Norman.
Misconception: Students often confuse the Design Model with the System Image, or think the System Image is only the graphics.

What cognitive concept explains this failure?

### Scenario

A home security application uses an icon of a key 🔑 to represent the "Delete Account" setting. User testing shows that many users click this icon hoping to change their password, and get frustrated.

### Answers

A. A mismatch between the user's mental model and the design model, caused by violating standard UI conventions.

B. A failure of working memory capacity because the key icon requires too many chunks of information to process.

C. A bottom-up perceptual processing error where the user fails to detect the color contrast of the key icon.

D. An absolute physical constraint mismatch that prevents the user's screen from rendering the key shape.

Correct: A
Explanation: הכשל נובע מכך שסמל המפתח מעורר אצל המשתמשים מודל מנטלי של אבטחה או סיסמה (מוסכמת עיצוב מוכרת). שימוש בסמל זה למטרה שונה לחלוטין (מחיקת חשבון) סותר את המודל המנטלי של המשתמשים ומייצר פער קשה מול מודל העיצוב. שאר האפשרויות מציעות כשלים קוגניטיביים שאינם רלוונטיים להבנת משמעות האייקון.
Learning Objective: Analyze usability issues caused by mental model mismatches.
Misconception: Students may attribute the error to memory capacity limits rather than semantic icon association.

How does top-down processing explain this failure?

### Scenario

A banking website places its main search input box in the bottom footer of the homepage instead of the header. User testing indicates that users fail to find the search input.

### Answers

A. Users scan the header because they expect the search convention to be there, filtering out other page areas.

B. Users suffer from working memory overload when scanning more than five distinct sections of a homepage.

C. Users fail to perceive the search input due to poor visual color contrast in the bottom footer layout.

D. Users require formal system training to understand the semantic utility of a search engine functions.

Correct: A
Explanation: עיבוד מלמעלה-למטה (Top-Down) נשען על ידע וציפיות מוקדמות (המודל המנטלי). מאחר שמוסכמת העיצוב המוכרת ביותר היא ששדה חיפוש ממוקם בחלק העליון (Header) של האתר, המשתמשים מסתכלים לשם ומצפים לראות אותו. כשהוא ממוקם בתחתית, הציפייה שלהם גורמת להם להתעלם מאזורים אחרים ולפספס אותו. עומס בזיכרון או ניגודיות אינם מסבירים כשל זה.
Learning Objective: Explain the connection between top-down processing, UI conventions, and mental models.
Misconception: Students sometimes blame visual styling (like contrast) instead of cognitive expectation and conventions.
